Problem Statement

Core Problem

The dependence on internet connectivity in current academic resource sharing systems poses significant barriers to effective learning in Nigerian universities.

Specific Challenges:

  • Students and lecturers frequently lose access to essential materials due to poor internet connections
  • Existing platforms designed for well-connected environments
  • Manual server configuration requirements
  • Inefficient methods: email attachments, USBs, physical handouts
[Problem Impact Visualization]

Research Evidence:

  • 78% of Nigerian universities face regular connectivity disruptions
  • 45% report significant impact on teaching/learning activities
  • 67% experienced significant disruptions to cloud-based platform access

Research Gap:

No comprehensive offline-first solution exists combining user-friendly design, automatic server discovery, and role-based access control for connectivity-constrained environments.

Aim and Objectives

AIM

To design, develop, and implement VelocityVer - an offline-first academic resource sharing application tailored for university environments with limited internet access.

OBJECTIVES:

1. DESIGN

Design an offline-first Android mobile application using Flutter framework with automatic server discovery and role-based access control

2. DEVELOP

Develop a lightweight Python Flask server architecture enabling secure local network resource sharing with SQLite offline storage

3. EVALUATE

Implement and test the system to evaluate functionality, performance, and user acceptance in real-world academic settings

[System Architecture Overview]

Technology Stack:

  • Mobile: Flutter 3.16.0 with Dart 3.1.0
  • Server: Python 3.11.0 with Flask 2.3.0
  • Database: SQLite 3.42.0
  • Architecture: RESTful API communication

  • Role-based content prioritization

  • UDP broadcasting protocol

Role-Based Access Control
